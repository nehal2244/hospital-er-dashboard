🏥 Hospital ER Analytics Dashboard

This project is an attempt to turn raw hospital data into something meaningful and useful. Using 9,216 Emergency Room patient records, I built an interactive dashboard that helps understand what’s really happening inside an ER — from patient flow to wait times and critical cases.

🚀 Project Overview

Hospitals deal with a huge amount of data every day, but without proper analysis, it’s hard to make quick and effective decisions.

In this project, I converted that raw data into a clean, easy-to-understand dashboard that highlights key patterns, bottlenecks, and performance gaps.

What makes this interesting is that the entire process — from cleaning the data to building the dashboard — was done with the help of Generative AI, making the workflow faster and more efficient.

📊 Key Features
Clear KPI cards showing important metrics like total patients, average wait time, and admission rate
Multiple interactive charts for better understanding of trends
Easy-to-use filters (slicers) such as:
Department
Patient severity
Gender
Age group
Admission status
Date range
A heatmap to identify the busiest hours and days
Visual breakdown of wait times and critical cases
📈 Key Insights
Around 50% of patients were admitted, which may indicate a heavy load on hospital resources
Neurology department showed the highest wait time (36.8 minutes), pointing to a possible bottleneck
Nearly 58% of patients were walk-ins, suggesting gaps in primary healthcare access
💡 Business Impact

This dashboard helps answer important questions like:

When does the hospital need more staff?
Which departments require attention?
How can patient wait times be reduced?

Overall, it provides a clear direction for improving efficiency and patient experience.
